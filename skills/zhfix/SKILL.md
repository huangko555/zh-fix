# zhfix skill

用户触发了 `/zhfix`(或自然语言说要改某文件的中文标点)。任务:对**单个文件**应用 zh-fix 规则,先备份再直接改,事后给个恢复入口。

## 硬约束(必须遵守)

1. **必须有具体文件路径。** 用户没明说哪个文件 → 问"想改哪个文件? 给我个路径。",不要自己猜"当前打开的"。
2. **不接受目录、通配符、"所有"。** 给了 `docs/` 或 `*.md` → 拒绝,要求指定单个。
3. **文件类型必须 `.md/.markdown/.html/.htm`。** 别的拒绝。
4. **改前必须备份。** 备份成功才能改。备份失败 → 停下报错,不要继续。
5. **不做 yes/no 确认。** 直接改,用备份兜底。
6. **范围跨过 markdown 结构边界**(比如截到半个代码块、半个 HTML 块) → **警告**用户,问是否仍要继续。

## 步骤

### 1. 解析

- `$FILE` = 用户给的路径(转成绝对路径)
- `$RANGE` = 用户提到的范围(可能是行号 `L20-L40`、章节 `## 用户故事`、段落 `第三段`、特定文本)
  - 没提范围 → `$RANGE = "full"`(整文件)

### 2. 校验

- 文件存在:不存在 → 拒绝
- 扩展名在 `.md/.markdown/.html/.htm`:否则拒绝

### 3. 找工具位置

```bash
cat ~/.zhfix/config.json
```

读 `tool_root` 字段。工具入口 = `<tool_root>/zh-fix.mjs`。

config 不存在 → 告诉用户 zh-fix 还没装,看 README 装好再来。

### 4. 备份(无条件先做)

```bash
mkdir -p ~/.zhfix/backups
TIMESTAMP=$(date +%Y%m%d-%H%M%S)
# 编码:把绝对路径里的 / \ : 都换成 _,确保不同目录下同名文件不冲撞
ENCODED=$(echo "$FILE_ABS" | sed 's/[\/\\:]/_/g')
BACKUP_PATH="$HOME/.zhfix/backups/${ENCODED}.${TIMESTAMP}.bak"
cp "$FILE_ABS" "$BACKUP_PATH"
```

记下 `$BACKUP_PATH`,后面要在报告里给用户。

### 5. 处理

#### 5a. `$RANGE === "full"`(整文件)

直接对原文件跑工具(zh-fix.mjs 会做 atomic write):

```bash
node "<tool_root>/zh-fix.mjs" "$FILE_ABS"
```

#### 5b. `$RANGE` 是某段范围

1. **识别行号** `L_start, L_end`:
   - 行号区间:直接用
   - 章节:Read 文件,找 `## XXX` 那行 = L_start;往下找下一个同级或更高级 heading 之前一行 = L_end
   - 段落:按空行分割,数到第 N 段
   - 特定文本:Grep 找该文本所在行,前后扩到段落边界

2. **边界检查**:扫描 L_start 之前是否有未闭合的 ``` ``` `` 围栏、`<script>`、`<style>` 等。如果有 → 警告"范围切到了半个结构,可能 mask 不准。继续吗?"

3. 抽出那段写到临时文件(扩展名同原文件):

```bash
TMP="/tmp/zhfix-range-$$.<同原扩展名>"
sed -n "${L_start},${L_end}p" "$FILE_ABS" > "$TMP"
```

4. 对临时文件跑工具:

```bash
node "<tool_root>/zh-fix.mjs" "$TMP"
```

5. 读处理后的临时文件,用 Edit 工具替换原文件 L_start..L_end 那几行

6. 删临时文件

### 6. 报告

```
✅ 已改:<file 路径>
范围:整个文件 / L20-L40(第三段)/ ## 用户故事 那节
改动:N 处(可以举 1-2 个 before→after 例子,从 backup 和当前文件 diff 出来)

原文件已备份,不满意可以 `zhfix restore "<file 路径>"` 还原。
```

**不要**告诉用户备份的具体路径(他们用 `zhfix restore` 就够了,知道路径反而是噪音)。

## 别做的事

- ❌ 不要先 diff 后问 yes/no(直接改,用备份兜底)
- ❌ 不要批量(多文件 / 目录 / 通配符)
- ❌ 不要在范围解析不清时硬猜(问)
- ❌ 不要跳过备份(备份失败必须停)
- ❌ 不要触发不属于"标点"的请求(改错字、改结构、改样式等都不是)
